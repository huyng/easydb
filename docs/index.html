EasyDB
Quick start API Config GitHub
Go · SQLite · Zero dependencies

SQLite databases,
served over HTTP.

EasyDB is a single Go binary that turns any SQLite file into a REST API — auto-generated CRUD, raw SQL, scheduled backups, and a built-in admin UI. No config files. No dependencies.

Get started View source
bash
$ git clone https://github.com/your-org/go-easydb && cd go-easydb
$ go build -o easydb ./cmd/easydb/
$ ./easydb --db mydata.db
 
EasyDB listening on http://127.0.0.1:8000
Admin UI: http://127.0.0.1:8000/admin/
Registered: mydata (mydata.db)

What you get

Everything, out of the box.

🗄️
Multi-database
Register and query multiple SQLite files from a single server.
⚡
Auto CRUD
Row-level GET, POST, PUT, DELETE generated for every table automatically.
🔍
Raw SQL
Execute arbitrary parameterised queries. SELECT returns rows; DML returns rowcount.
💾
Backups
On-demand or scheduled snapshots via VACUUM INTO. Auto-rotation and one-click restore.
🖥️
Admin UI
Built-in browser interface for browsing tables, running SQL, and managing backups.
🔑
API key auth
Set API_KEYS to require an X-API-Key header. Multiple keys supported.
🛡️
Safe by default
Identifiers double-quoted, values parameterised. WAL mode. SSRF protection on imports.
📦
Single binary
Pure Go. No runtime dependencies. Admin UI embedded in the binary at build time.

Quick start

Up in three steps.

1
Build

Requires Go 1.22+. Clone the repo and build the binary with a single command.

2
Run

Start the server. Use --db to register a SQLite file on startup, or add databases via the API.

3
Query

Hit the REST API directly, or open the Admin UI at /admin/ for a browser interface.

4
Configure

Set API_KEYS, BACKUP_SCHEDULE, and other options via environment variables or a .env file.

install & run 
# clone and build
git clone https://github.com/your-org/go-easydb
cd go-easydb
go build -o easydb ./cmd/easydb/

# optional: install to PATH
mv easydb /usr/local/bin/

# start with a database
easydb --db mydata.db
query the API 
# list tables
curl -H "X-API-Key: $KEY" \
  localhost:8000/api/databases/mydata/tables

# insert a row
curl -X POST -H "X-API-Key: $KEY" \
  -H "Content-Type: application/json" \
  -d '{"name":"Alice","age":30}' \
  localhost:8000/api/databases/mydata/tables/users/rows

API reference

Every endpoint.

Databases
GET/api/databasesList all
POST/api/databasesRegister
POST/api/databases/uploadUpload .db file
POST/api/databases/importImport from URL
POST/api/databases/backup-allBackup all
DELETE/api/databases/{db}Unregister
GET/api/databases/{db}/infoMetadata
Tables & Schema
GET/api/databases/{db}/tablesList with row counts
GET/api/databases/{db}/tables/{table}/schemaColumn schema
GET/api/databases/{db}/tables/{table}/ddlCREATE statement
Rows
GET/api/databases/{db}/tables/{table}/rowsList (paginated)
POST/api/databases/{db}/tables/{table}/rowsInsert
GET/api/databases/{db}/tables/{table}/rows/{id}Get by PK
PUT/api/databases/{db}/tables/{table}/rows/{id}Update (partial)
DELETE/api/databases/{db}/tables/{table}/rows/{id}Delete
Raw SQL
POST/api/databases/{db}/queryExecute SQL
SELECT returns columns + rows, capped at 10 000 rows. DML/DDL returns rowcount + lastrowid. Truncated responses include "truncated": true.
Backups
POST/api/databases/{db}/backupCreate
GET/api/databases/{db}/backupsList
GET/api/databases/{db}/backups/{file}Download
POST/api/databases/{db}/backups/{file}/restoreRestore
DELETE/api/databases/{db}/backups/{file}Delete

Configuration

All via env vars.

Variable Default Description
API_KEYS — Comma-separated API keys. If unset, runs in open dev mode.
DATA_DIR data Directory for databases and the registry.
ADMIN_ENABLED true Set to false to disable the admin UI.
CORS_ORIGINS * Comma-separated allowed origins. Use specific origins in production.
HOST 127.0.0.1 Bind address.
PORT 8000 Port number.
BACKUP_DIR data/backups Directory for backup files.
BACKUP_MAX_COUNT 5 Max backups per database before oldest are auto-deleted.
BACKUP_SCHEDULE — Auto-backup interval: 30m, 6h, 1d, etc.
EASYDB_OPEN — SQLite file to auto-register on startup (same as --db).
CLI flags
--db         string   SQLite file to auto-register on startup
--host       string   Bind address      (overrides HOST)
--port       string   Port number        (overrides PORT)
--data-dir   string   Data directory     (overrides DATA_DIR)
